Working hexo with dropbox and cloud9

Posted on 2015-11-24   |  

Finally, I know how to sync files with Dropbox in cloud9, so I can edit .md file at my computer and generate the website in cloud9 which sounds perfect for me. The steps also work with Koding or other cloud based coding platform which have a Linux environment.

I adopt the main steps form help in cloud9 https://c9.io/blog/dropbox-on-cloud9/

  1. Assume you already have a dropbox account and install it in your own computer

  2. Install Dropbox on Cloud9 from the terminal

    $ cd ~ && wget -O - "https://www.dropbox.com/download?plat=lnx.x86_64" | tar xzf -

  3. Start the initial linking of your Dropbox account and synchronization of files.
    Note: After you run this command, it will repeatedly tell you, “This computer isn’t linked to any Dropbox account…”. Ignore that and carry on to step 5

    $ ~/.dropbox-dist/dropboxd

  1. Click the link to verify which account you want to connect.

    After that, the terminal should say “This computer is now linked to Dropbox.” Now type Ctrl + C to stop the script because it is synchronizing all your files in dropbox.

  2. Create a symlink for your new Dropbox folder in your workspace folder

    $ ln -s /home/ubuntu/Dropbox/ ~/workspace/

Note: You may toy with doing things the other way around and creating a symlink to
your workspace folder inside the Dropbox folder like this:

$ ln -s ~/workspace/ ~/Dropbox/myCloud9Folder

This is important. In fact, I use

ln -s ~/workspace/blog/source ~/Dropbox/Entertainment/Hexo

If it is wrong, you can also unlink it and do it again.

unlink source    (in ~/Dropbox/Entertainment/Hexo)

Once your Dropbox account is linked and synced, you’ll want access to sync settings (like auto sync) and the ability to easily run Dropbox commands in the future. To do this, download the Dropbox script and add it to your PATH.

  1. Change to your home directory and download the Dropbox python script

    $ cd ~/ && wget https://linux.dropbox.com/packages/dropbox.py

  2. Make your Dropbox script executable

    $ chmod +x dropbox.py

  3. Add the Dropbox script to you PATH

    $ cd /bin && sudo ln -s ~/dropbox.py

  4. Start the syncing script

    dropbox start

    Otherwise, you can stop it by

    dropbox stop

  5. Prevent Dropbox from syncing files and folders by adding them to your exclude list

    dropbox exclude add ~/Dropbox/Useful

    Once the directory is excluded for syncing, the files already downloaded in that directory will also be removed in that computer, but it remains in other computer.

    You can also use the command to see whether it works

    dropbox exclude list
